# CLAUDE.md

본 레포에서 작업을 시작할 때 다음 두 문서를 먼저 읽고, 모든 코드 작성·수정은
두 문서를 기준으로 한다.

- [`tree.md`](./tree.md) — 디렉토리 구조와 각 폴더의 역할.
  파일을 추가하기 전에 어디에 속하는지 확인한다.
- [`rule.md`](./rule.md) — 코딩 규칙. 곁가지 기능 배제, 학부생 수준
  가독성, 주석에 이모지 금지 등.

## 프로젝트 컨텍스트

서버 예약/할당 관리 시스템의 **서버 풀 에이전트** 레포다.
백엔드(APScheduler)가 메트릭을 수집해갈 대상 서버들을 Docker 컨테이너로
시뮬레이션한다. 한 종류의 경량 에이전트 이미지를 N개 띄워 N대의 서버를
흉내내며, 각 컨테이너는 `:9101/metrics` 엔드포인트를 노출한다.

설계 원칙상 백엔드와 의존성을 공유하지 않으며 에이전트는 가볍게 유지한다.

전체 시스템 설계(유스케이스, 아키텍처, 기술 스택)는 별도 `diagram-and-docs`
레포의 `index.html`·`use-case-spec.html`·`project-plan.html`·`tech-stacks.html` 을 참조한다.

관련 레포:

- `backend` — FastAPI + APScheduler (본 에이전트로부터 메트릭 수집)
- `frontend` — React SPA

## 작업 시 체크

- 새 파일 생성 위치가 `tree.md` 구조에 맞는가.
- 디렉토리 구조를 바꿔야 한다면 코드보다 `tree.md` 를 먼저 갱신했는가.
- 작성한 코드가 `rule.md` 의 5개 규칙을 모두 통과하는가.
- 메트릭 JSON 스키마·엔드포인트 규약을 본 레포에서 임의로 바꾸지 않았는가
  (단일 출처는 설계 문서).
